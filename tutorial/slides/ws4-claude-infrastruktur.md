# Workshop 4: Context Engineering und Claude-Infrastruktur

> Context Engineering, Claude Desktop als Arbeitsumgebung und die Anthropic-Produktlandschaft – der vierte Workshop im Legal History Hub Projekt.

**Datum:** 12.05.2026
**Dauer:** 4 Stunden (Remote)
**Leitung:** Christian Steiner

## Themen

- Von Prompt Engineering zu Context Engineering: warum Frontier Models gut kuratierten Kontext brauchen
- Drei Schichten des Kontexts: System-Instruktionen, Projekt-/Session-Kontext, Einzel-Prompt
- Context Window und Context Rot: warum mehr Tokens nicht automatisch besser sind
- Drei Kernoperationen des Context Engineering: Creation, Compression, Orchestration
- Distillation: Markdown als Arbeitsformat (token-effizient, versionierbar, LLM-nativ)
- Wissensdokumente als materialisierte Context Compression
- Critical Expert in the Loop: Sycophancy, Konfabulation, Bias und Prüffragen
- Claude Desktop jenseits des Chats: Projects, Artifacts, Memory, (Custom) Skills
- Connectors und MCP (Model Context Protocol): native Integrationen und der offene Standard
- Plugins als Bundle aus Skills, Subagents und Connectors (verfügbar in Cowork und Code)
- Datenschutz und DSGVO: Training-Opt-out, DPA, Zero Data Retention, Verarbeitungsort (US vs. EU)
- Die Claude-Produktfamilie: Claude.ai, Desktop, Mobile, Office, Chrome, Cowork, Code, Design
- Token-Ökonomie: Plan-Realität, Wochen-Caps, Modellwahl Sonnet vs. Opus
- Use Cases live: PowerPoint-Generierungswege, Excel, Meeting-Protokolle, Jour-Fixe-Aggregation, Claude in Chrome
- Workflow für dieses Deck: Claude Code in VS Code → DHCraft Apps Script Importer → Google Slides
